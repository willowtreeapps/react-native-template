# Why We Use React Query

This project standardizes server-state on TanStack Query with a single app-level `QueryClient` provided from `AppProviders`, while screens and components consume domain hooks such as `useHomeQuery` instead of embedding fetch logic directly. We chose TanStack Query over lighter alternatives because this project is intended to support apps of any size, from simple prototypes to enterprise-grade critical applications, and it provides high-value capabilities such as caching, offline support, and automatic refetching. We made this decision to provide a production-ready data foundation from Day One that reduces duplication, keeps behavior consistent across features, and gives teams and agents a clear testing seam through hook mocking and provider-backed test rendering.
